# YAGNI — You Ain't Gonna Need It

Follow these principles when writing or reviewing code:

1. **Don't add code "just in case"** — only implement what is required now.
2. **Prefer simple solutions** — don't abstract or generalize before there's a clear, repeated need.
3. **Avoid premature optimization** — write clear code first, optimize only when profiling shows it's needed.
4. **Question speculative features** — if a feature isn't in the current requirements, don't build it.
5. **Keep the codebase small** — every line of code is a liability (to read, test, debug, and maintain).
